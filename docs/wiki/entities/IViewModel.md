# IViewModel

> The core contract every ViewModel implements. You almost never write it by hand — `[ViewModel]` generates it. See [[ViewModel Generation]].

## Surface

```csharp
public interface IViewModel
{
    FindBindableMemberResult FindBindableMember(in FindBindableMemberParameters parameters);
}
```

A single method: given parameters identifying a member, return a result describing the bindable member found. This is the lookup [[Binders Catalog|binders]] use to resolve which ViewModel member they connect to.

## Why it's shaped this way

`in FindBindableMemberParameters` is passed by readonly-reference to avoid copying a parameter struct on every lookup — consistent with the framework's no-reflection, low-allocation goal. The generated implementation resolves members without runtime reflection.

## Key relationships

- Generated by `[ViewModel]` → [[ViewModel Generation]].
- Consumed by binders at View initialization ([[View Initialization]]) and during [[Runtime Binding Resolution]].
- Related contracts: [[IRelayCommand]], [[IBinder]].
